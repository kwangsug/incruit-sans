---

## 1. 프로젝트 배경

Incruit Sans v0.1.0은 Pretendard 베이스 리브랜드 + tnum 기본화로 출시 완료. v0.2는 Latin 글리프를 **Source Sans 3 (Adobe)**로 교체하여 채용 문서에 더 적합한 톤(전문적·따뜻함)을 확보한다.

**v0.1 → v0.2 차이의 본질:**
- v0.1: Pretendard의 Latin (Inter 계열) 그대로 → 모던 UI 톤
- v0.2: Source Sans 3 Latin 교체 → 전문 문서 톤 + 본고딕(Source Han Sans) 메트릭 호환

---

## 2. 작업 범위

### 2.1 핵심 작업: Latin 글리프 병합

**대상 Unicode 범위 (필수):**
- Basic Latin: U+0020 ~ U+007E (95자)
- Latin-1 Supplement: U+00A0 ~ U+00FF (96자)
- Latin Extended-A: U+0100 ~ U+017F (128자, 유럽 외국인 이름)
- General Punctuation 일부: U+2000 ~ U+206F (각종 dash, quote)
- Currency: U+20A0 ~ U+20CF (€, ₩, $ 등)

**대상 Unicode 범위 (선택):**
- Latin Extended-B: U+0180 ~ U+024F
- Cyrillic: U+0400 ~ U+04FF (외국인 이름, 옵션)

### 2.2 기술 챌린지

| 항목 | 상세 |
|------|------|
| em 사이즈 | Source Sans 3 (1000) → Pretendard (2048) 스케일링 (× 2.048) |
| 글리프 매핑 | Source Sans 3 ↔ Pretendard 글리프 이름 규칙 다름, codepoint 매칭 필요 |
| Vertical metrics | Pretendard ascent 1949 / descent -494 vs Source Sans 1000/-326 — Pretendard 메트릭 유지 |
| x-height | 한글 글리프와 자연스러운 균형 (필요시 미세 조정) |
| Hinting | OTF (CFF) hinting 재계산 — 9pt 인쇄 가독성 보장 |
| Kerning | Source Sans 3 GPOS kern 재구성 → Pretendard GPOS와 통합 |
| OpenType features | tnum, lnum, onum, pnum, frac, smcp, ss01, ss02 모두 동작 보장 |

### 2.3 제외 (v0.3 이후)

- 한글 자체 디자인 변경 (Pretendard 한글 그대로 유지)
- 한국 punctuation 자체 디자인 (`·`, `~`, `–`, `※` 등) — v0.3 작업
- Stylistic Sets ss01/ss02 정의 — v0.4 작업
- 한자 글리프 추가 (영구 미포함, fallback 위임)
