# 폰트 엔지니어 모십니다 — Incruit Sans 프로젝트

> **한국 채용 문서를 위한 첫 번째 표준 폰트, Incruit Sans v0.2를 함께 만들 폰트 엔지니어를 찾습니다.**

---

## 우리는

인크루트(Incruit)는 1998년 창업한 한국 1세대 채용 플랫폼입니다. 27년간 수천만 건의 이력서를 처리해왔고, 이제 한 가지 단순한 질문에 답하려 합니다.

**"왜 한국에는 이력서를 위해 디자인된 폰트가 없는가?"**

Pretendard는 모던 UI에 적합하고, Wanted Sans는 채용 브랜드 폰트지만, **이력서·자소서·포트폴리오라는 채용 문서 자체를 위해 설계된 폰트는 없습니다.** 인크루트가 그 첫 번째를 만들고, OFL 1.1로 공개합니다.

**Incruit Sans 차별화:**
1. `tnum` 기본 활성화 — 이력서 표가 자동으로 정렬됨
2. 한국 이력서 punctuation 시각 보정 (`·` `~` `–` `※`)
3. 9pt A4 인쇄 가독성 인증
4. ATS 파싱 호환 인증 (인크루트·사람인·잡코리아)
5. Stylistic Sets로 신입(표현적) / 임원(보수적) 톤 분기

v0.1은 출시 완료(Pretendard 베이스 리브랜드 + tnum 기본화). 이제 **v0.2: Latin 글리프를 Source Sans 3 (Adobe)로 교체**할 엔지니어가 필요합니다.

## 무엇을 만드시나요

### 핵심 작업

**Source Sans 3 (em 1000) → Pretendard (em 2048) Latin 글리프 병합**

- 대상: Basic Latin + Latin-1 Supplement + Latin Extended-A + 일부 punctuation/currency (약 350자)
- em 사이즈 정합 (× 2.048 스케일링)
- Pretendard의 한글·메트릭·GSUB/GPOS 구조 보존
- Source Sans 3의 GPOS kern 통합
- OTF (CFF) hinting 재계산 — 9pt 인쇄 가독성 보장

### 산출물

- `IncruitSans-{Light,Regular,Medium,Bold}.otf` 4종 + WOFF2 변환
- `scripts/build_v0.2.py` — 재현 가능한 빌드 자동화
- 9pt 인쇄 검증 리포트 (A4 5장)
- ATS 파싱 호환 테스트 리포트
- v0.3 작업 인계 문서

## 이런 분을 찾습니다

### 필수

- **OpenType / CFF 폰트 구조 이해** — name table, cmap, GSUB/GPOS, CFF charstring 직접 조작 경험
- **fontTools (Python) 또는 FontForge / Glyphs / RoboFont 능숙**
- **다른 em 사이즈 폰트 병합 실적** (글리프 스케일링 + 메트릭 정합)
- **한글 폰트 작업 경험** — Pretendard, 본고딕, Spoqa Han Sans, SUIT 등 contributor 또는 fork 경험

### 우대

- ✨ Pretendard repo contributor 이력
- ✨ Adobe Source 시리즈 또는 Google Noto 폰트 작업 이력
- ✨ 산돌·윤디자인·AG타이포그라피 출신
- ✨ ATS / PDF 임베딩 호환성 디버깅 경험
- ✨ Variable Font 구축 경험 (v1.0 작업 시 추가 발주)

### 이런 분께는 적합하지 않습니다

- 일러스트레이터로 시각 디자인만 가능, 폰트 엔지니어링 경험 없음
- TTF만 다뤄봄 (CFF 미경험)
- 폰트 라이선스(OFL) 처리 경험 없음
